■ ネットワーク層
IP　　　　インターネットプロトコルのこと　L3で使われる

IPヘッダ　L3で貼り付けられるやつ、パケットの先頭のやつ

ICMP　　　Internet Control Message Protocol　ネットワーク通信の問題を診断するために
　　　　　　ネットワークデバイスが使用するネットワーク層のプロトコル　　

ICMPメッセージ　　送信したデータが受信されたか、受信された場合タイムリーに届いたかなど、通信状況を確認する為のやつ

IPアドレス　　　　L3で使われるアドレスのこと

オクテット　　　　アドレスで.や：で区切られている一つずつの括りのことをオクテットという。
　　　　　　　　　また、先頭から第一オクテット、第二オクテット などと呼ばれる。

ネットワーク部　　そのIPアドレスが属しているネットワークを識別する為の部分。

ホスト部　　　　　ネットワーク内のコンピューターを識別するための部分。

クラス　　　　　　A,B,C、Dとあり、一番先頭が０や１０、１１０などによって分けられる。

プライベートIPアドレス　特定のネットワーク内で割り当てられるIPアドレスの事。「ローカルIPアドレス」ともいう
　　　　　　　　　　　　家の中や社内などの限定されているネットワーク内

グローバルIPアドレス　　インターネットに接続するときに使われる。ダブってはいけないから管理されている。
　　　　　　　　　　　　プライベートIPアドレスからグローバルIPアドレスに通信するときはルーターが変換してくれる。

ネットワークアドレス　　ホスト部が全て０のやつ

ブロードキャストアドレス　ホスト部が全てフル満のやつ

ループバックアドレス　　　そのコンピューター自身を示すIPアドレスの事。OSなどに実装された仮想的なNICである
　　　　　　　　　　　　　ループバックインターフェイスに割り当てられる。

サブネット　　　　　　　　ネットワーク内の小規模ネットワークの事。

サブネットマスク　　　　　ネットワークの範囲を定義するために使用される。

プレフィックス表記　　　　サブネットマスクがどこまで１ってのを短く表記したやつ

デフォルトゲートウェイ　　ルータが他のネットワークにつなぐためにある入り口的なもの

IPv6　　　　　　　　　　　IPv4よりすげーやつ。340巻ある。128ビットで構成される。：を使うやつ

サブネットプレフィックス　　基本前半64ビットのやつ

インターフェイスID　　　　　ホスト部的なやつ。MACアドレスの間にFFFEを入れて上から7ビット目を二進数的に入れ替えてやるやつ

ルータ　　　　　　　　　　　他のネットワークに接続するために使われるやつ。L3のやつ

コンソール接続　　　　　　　最初にスイッチを設定するために使用される　きしめん

VTY接続　　　　　　　　　　「virtual teletype」ルータなどの通信機器にネットワークを通じて別のコンピュータから接続し、管理や操作を行う為の仮想的な接続口

AUX接続　　　　　　　　　　モデムを通じて公衆回線を利用して機器にアクセスする方法

ルーティングテーブル　　　　ルータ内での経路情報のこと。どこから何処を通れば効率的かを覚えている表的なやつ

ネクストホップ　　　　　　　パケット転送する際に次に転送されるルータまたはノードのアドレスを指す

ARPテーブル　　　　　　　　ARPによって割り出したIPアドレスとMACアドレスの対応表

ARP　　　　　　　　　　　　「Address Resolution Protocol」IPアドレスからMACアドレスを求める為のプロトコル

コアルータ　　　　　　　　　通信キャリアやISPネットワークの通信を通経する為に使われるルータ。大量のネットワークトラフィックを転送する必要があるため、
　　　　　　　　　　　　　　高速なパケット転送能力と高い可用性が求められます。

センタールータ　　　　　　　企業の拠点間WANネットワークで本社拠点・データセンタ等、企業で中心的に利用されるネットワークを接続するために利用される

エッジルータ　　　　　　　　企業の拠点間WANネットワークで支社や営業所等の拠点ネットワークを接続するために利用される

ブロードバンドルータ　　　　家庭環境やSOHO環境でインターネット回線に接続する目的で利用される

ルーティング　　　　　　　　ネットワーク上でデータを転送する際経路を導き出すこと

直接接続　　　　　　　　　　ルータが直接接続しているネットワークのルートのこと

スタティックルーティング　　管理者がネットワークへの最適なルートを手動で設定したルートのこと

ダイナミックルーティング　　ルーティングプロトコルにより動的に追加されるルートのこと

ディスタンスベクタ型　　　　ルーティングプロトコルの特徴・分類の一つで隣接するルータ間でルーティングテーブル（経路表）を交換し合い、どの隣接ルータを経由すれば最短のホップ数で宛先に届くかを基準に経路を指定する方式

リンクステート型　　　　　　ネットワーク上の全てのルーターが持つネットワークの情報を、全ルーターが共有し合うことによって最適なルートを選択するルーティングプロトコルです。

ハイブリッド型　　　　　　　Ciscoが独自にディスタンスベクタ型を拡張したルーティングアルゴリズムです。 ベースはディスタンスベクタなので、ルータ同士が交換する情報は、
　　　　　　　　　　　　　　ネットワークアドレス/サブネットマスク+メトリックです。 ただし、定期的ではなく、ネットワークに変更があったときのみ交換します。

 
■ トランスポート層

コネクション型通信　　　　　信頼性を重視する手続きを行う通信　TCP

ACK　　　　　　　　　　　　わかったよーってやつ

コネクションレス型通信　　　効率重視の手続きを行う通信　UDP

TCP　　　　　　　　　　　　コネクション型通信プロトコル

スリーウェイハンドシェイク　SYN→ACK+SYN→ACKのやつ　TCPの最初に通信するときのやつ 

Telnet　　　　　　　　　　　23番　TCP 暗号化できない　機器をリモート操作できる

HTTP　　　　　　　　　　　　80番　TCP ウェブブラウザと通信する際に用いられる

FTP　　　　　　　　　　　　　20、21番　TCP　ファイルの転送、制御のやつ

SMTP　　　　　　　　　　　　25番　TCP メールを送信

TCPヘッダ　　　　　　　　　　セグメント　めっちゃデータパチパチ

UDP　　　　　　　　　　　　　コネクションレス型通信プロトコル 

RTP　　　　　　　　　　　　　RTPの特徴 一般的なRTPパケットの構造を(図6)に示します。 RTPの大きな目的は符号データの
　　　　　　　　　　　　　　　フレーミング(パケット化)
　　　　　　　　　　　　　　 送信側のパケット送出タイミングを受信側でも復元できるようにすることです。

SNMP　　　　　　　　　　　　161番　UDP　ネットワーク機器の管理監視

DHCP　　　　　　　　　　　　67、68番　UDP IPアドレスなどの情報を自動的に割り当てる

TFTP　　　　　　　　　　　　69番　UDP 認証を行わないファイル転送

RIP　　　　　　　　　　　　520番　ダイナミックルーティング の一種

UDPヘッダ　　　　　　　　　データグラム　TCPヘッダより小さい
